# TASK-2026-03-05-A: Restore Shift-Click Force Start + Force-Start On-Time Credit

**Created:** 2026-03-05  
**Recommended Executor:** Codex  
**Scope:** Daily schedule/session start behavior (single atomic fix)  
**Flags:** `IMPLEMENT UNCOMMITTED, VERIFY FIRST`

---

## Goal

Restore shift-click force start for scheduled sessions, and when shift-click is used outside the normal start window, treat that session as on-time for obligation/schedule completion.

---

## Problem Statement

Previously, shift-click on a scheduled session could bypass the time-window lock and launch practice. That behavior is now failing/regressed in runtime behavior.  

Additionally, when force-starting outside the scheduled time, completion should be credited as if started on time (for schedule obligation matching), while preserving real session timing for analytics/duration.

---

## Single Hypothesis

The shift-click signal is not being carried end-to-end from start surface to session recording semantics, so either launch bypass fails or forced launches are recorded with out-of-window timing (no matched obligation credit).

---

## Single Atomic Change

Add one explicit force-start path-context signal and consume it in session recording so:

1. shift-click reliably launches from locked slot/leg surfaces, and  
2. forced off-window starts are recorded with an on-time schedule match snapshot.

No unrelated refactors.
