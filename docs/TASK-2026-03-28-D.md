## Goal

CLAUDE.md requires testing at 320px, 375px, and 412px widths with 44px minimum touch targets. This task pins to **375px** as one required baseline and targets **AuthGate** — the first interactive surface every user passes through.

AuthGate uses inline `style={{ padding: 10 }}` on inputs and buttons. These are hypothesized to produce computed heights below the repo's 44px touch-target policy. This task measures first, then corrects only confirmed failures.

---

Potentially undersized interactive elements to measure at 375px:

| Element | Location | Hypothesis |

**These are hypotheses, not established facts.** The pre-implementation audit confirms or rejects each one.

## Implementation (Only for Confirmed Failures)

For each AuthGate element confirmed below 44px at runtime, apply the minimal inline-style correction needed. Prefer `minHeight: 44` first. Only increase padding where required to preserve appearance.

Default correction pattern:

```js
style={{ minHeight: 44, padding: 10, borderRadius: 10, ... }}
```

For elements where `minHeight` alone is insufficient (e.g., a link-styled toggle where visual padding is needed for the tap area):

```js
style={{ minHeight: 44, padding: "8px 4px", display: "inline-flex", alignItems: "center", ... }}
```

Do not change border-radius, colors, fonts, or layout flow. Only sizing.

Do not correct elements that already meet the 44px baseline, even if they are listed as hypotheses above.
