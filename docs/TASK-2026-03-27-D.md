## Goal

Several production code paths emit console output on every render cycle, every audio gain adjustment, and every avatar stage transition. None are gated by a dev flag. They bloat the console in production, expose internal state, and add overhead in hot paths — theme/stage transitions fire on every avatar breathing cycle during active sessions.

## Implementation

### Pattern: gate with `import.meta.env.DEV`, do not delete

Wrap each unguarded call with `if (import.meta.env.DEV)`. Do not alter log message content.

```js
// Before
console.log("[IsoEngine] setMasterGain clamped to:", v, "masterGain node:", this.masterGain);

// After
if (import.meta.env.DEV) {
  console.log("[IsoEngine] setMasterGain clamped to:", v, "masterGain node:", this.masterGain);
}
```

Apply this pattern to all 9 unguarded calls listed above. One change per call site — do not reformat surrounding code.

### Exception: `console.error` in genuine error handlers

Do **not** gate `console.error` calls that report genuine runtime errors (image load failures, network failures, etc.). Those belong in production.
