# TASK-2026-03-26-C: Bundle Split Phase 1 — Lazy-Load PracticeSection

**Created:** 2026-03-26
**Recommended Executor:** Claude Code / Codex
**Scope:** Convert `PracticeSection` from an eager import to a `React.lazy` dynamic import in `App.jsx`, following the established pattern already used for `WisdomSection` and `ApplicationSection`. Expected reduction: ~200 kB gzipped from the initial chunk.
**Flags:** `IMPLEMENT UNCOMMITTED, VERIFY FIRST` `NO COMMIT WITHOUT HUMAN APPROVAL`

---

## Goal

The main bundle is 4,671 kB uncompressed / 1,358 kB gzipped — well above the 500 kB warning threshold. `PracticeSection` is the largest contributor still loaded at startup. Deferring it behind a `React.lazy` boundary removes it from the initial chunk and reduces time-to-interactive for users who open the app on a non-Practice section.

This is Phase 1 of the strategy documented in `docs/BUNDLE_SPLIT_PLAN.md` (Strategy 1: "Lazy-load PracticeSection from app root").

---

## Problem Statement

`src/App.jsx` line 5:
```js
import { PracticeSection } from "./components/PracticeSection.jsx";
```

This is a synchronous named import. The entire `PracticeSection` module graph (including all breath rings, WebGL shaders, Three.js imports, and practice state) is included in the initial `index-*.js` chunk. The app pays this cost on first load regardless of which section the user navigates to.

`WisdomSection` and `ApplicationSection` are already lazy-loaded using the established pattern (App.jsx lines 10–11):
```js
const WisdomSection = lazy(() =>
  import("./components/WisdomSection.jsx").then(m => ({ default: m.WisdomSection }))
);
const ApplicationSection = lazy(() =>
  import("./components/ApplicationSection.jsx").then(m => ({ default: m.ApplicationSection }))
);
```

`PracticeSection` is a named export, so it requires the same `.then(m => ({ default: m.PracticeSection }))` wrapper.

---

## Single Bounded Change

Convert `PracticeSection` to a `React.lazy` import and wrap its render site in a `<Suspense>` boundary. No changes to `PracticeSection.jsx` itself, no store changes, no routing changes, no new files.

---

## Reuse-First Plan (Mandatory)

1. `src/App.jsx`
   - Remove line 5: `import { PracticeSection } from "./components/PracticeSection.jsx";`
   - Add a lazy declaration following the exact pattern of `WisdomSection`:
     ```js
     const PracticeSection = lazy(() =>
       import("./components/PracticeSection.jsx").then(m => ({ default: m.PracticeSection }))
     );
     ```
   - Locate the existing render of `<PracticeSection ...>` (around line 116). It is currently not wrapped in `<Suspense>`. Wrap it with a `<Suspense fallback={<div />}>` using the same minimal fallback pattern used for `WisdomSection` and `ApplicationSection`.
   - `lazy` and `Suspense` are already imported from React on line 2 — no new imports needed.

2. `src/components/PracticeSection.jsx`
   - **No changes.** The component already uses a named export. The lazy wrapper in App.jsx handles the named-to-default conversion.
