# TASK-2026-03-27-G: Remove HMR Probe Infrastructure from Production Bundle

**Created:** 2026-03-27
**Recommended Executor:** Claude Code / Codex
**Scope:** Remove or consolidate the avatar HMR probe scaffolding in `App.jsx` and `AvatarComposite.jsx`. This code is gated by `import.meta.env.DEV` at runtime but still ships in the production bundle, adding ~200 lines of dead weight and obscuring the host component logic.
**Flags:** `IMPLEMENT UNCOMMITTED` `NO COMMIT WITHOUT HUMAN APPROVAL`

---

## Background and Clarification

The original audit flagged `JSON.stringify` comparisons at `AvatarComposite.jsx` lines 351, 678–679. Upon investigation, these are **fully gated by dev guards** (`if (import.meta.env.DEV)` and `AVATAR_COMPOSITE_HMR_SUBSTRATE_PROBE_ENABLED = import.meta.env.DEV && Boolean(import.meta.hot)`). There is no production performance concern with those specific calls.

The actual concern is that both `App.jsx` and `AvatarComposite.jsx` contain substantial blocks of HMR probe scaffolding — probe context objects, logging functions, `window.__avatarHmrHostProbe__` mutation, mount/unmount instrumentation — that:

1. Ship in the production JS bundle (Vite only tree-shakes `import.meta.env.DEV` constants at build time if they're used as top-level `if` conditions — they are here, so this code **is** eliminated in production builds)
2. In **dev**, add significant noise to the component logic, making `App.jsx` and `AvatarComposite.jsx` harder to read and modify safely

Verify point 1 before proceeding: run `npm run build` and confirm neither `PROBE:avatar-hmr-host` nor `PROBE:avatar-hmr-substrate` strings appear in any `dist/` output file.

## Implementation (Dev-Clarity Path — Expected)

Assuming production bundle is clean and probes are no longer actively consulted:

### Option A: Remove probe blocks entirely (preferred if probes are resolved)

From `App.jsx`:
- Remove the entire `// PROBE:avatar-hmr-host:START` to `:END` block (lines 56–88)
- Remove `logAvatarHmrHostProbe` calls in `SectionView` `useEffect` (mount/unmount logging)

From `AvatarComposite.jsx`:

- Remove `logAvatarCompositeHmrSubstrateProbe` calls in the component body


This keeps the main component files clean while preserving the probe tooling.

**Choose Option A or B based on what you find in Step 2 of the read gate.** If uncertain, use Option B (safer).
