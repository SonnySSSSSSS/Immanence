## Goal

## Problem Statement

### Cause 1 — Root-relative paths bypass Vite `base` (all 7 images)

All seven `src` strings in `src/data/onboardingCurriculumContent.js` use root-relative absolute paths:

```js
src: '/tutorial/breath and stillness/breath tutorial 1.webp'
src: '/tutorial/breath and stillness/intensity 1.webp'
```

The leading `/` anchors to the server root. On GitHub Pages the app is deployed under `/Immanence/` (`vite.config.js` line 31). In production the browser requests `https://<user>.github.io/tutorial/...` — that path does not exist. The correct URL is `https://<user>.github.io/Immanence/tutorial/...`.

Every other working image reference in the app uses:

```js
`${import.meta.env.BASE_URL}tutorial/...`
```

`import.meta.env.BASE_URL` is `/` in dev and `/Immanence/` in production builds. This pattern is already used in `TutorialOverlay.jsx` (line 441), `RitualStepDisplay.jsx` (line 28 — with an explicit comment: "critical for GitHub Pages"), `SideNavigation.jsx`, `CompactStatsCard.jsx`, and many more. `onboardingCurriculumContent.js` is the only active production file that uses root-relative `/...` paths for static assets.

### Cause 2 — Four `breath tutorial` images are untracked and will not reach the deployed bundle

Git status shows these four files as untracked:

```
?? public/tutorial/breath and stillness/breath tutorial 1.webp
?? public/tutorial/breath and stillness/breath tutorial 2.webp
?? public/tutorial/breath and stillness/breath tutorial 3.webp
?? public/tutorial/breath and stillness/breath tutorial 4.webp
```

The three `intensity` images were committed in `744acb9`. The four `breath tutorial` images were never staged. Even after fixing the URL construction, these four would still 404 in production because the files are absent from the bundle.

### Cause 3 — `localStorage` overrides can re-introduce stale `src` strings after a code fix

`readOnboardingCurriculumContent()` merges `localStorage` overrides on top of compiled defaults (`mergeContent` in `onboardingCurriculumContent.js` lines 189–204). Override leaf values take priority over defaults (line 203). If any browser has a stored override that contains the old root-relative `src` string, the code fix does not help that user. Media paths are not a user-facing preference; they are code-owned. `src` fields must not be overridable from `localStorage`.

Replace each root-relative literal with a `BASE_URL`-relative path:

```js
// Before
```

### Sub-fix C — Strip `src` fields from `localStorage` override

Add a private helper before `readOnboardingCurriculumContent`:

```js
function stripSrcFields(obj) {
    if (!obj || typeof obj !== 'object') return obj;
    if (Array.isArray(obj)) return obj.map(stripSrcFields);
    const result = {};
    Object.keys(obj).forEach(key => {
        if (key === 'src') return; // src is code-owned, not user-overridable
        result[key] = stripSrcFields(obj[key]);
    });
    return result;
}
```

Call it in `readOnboardingCurriculumContent` before `mergeContent`:

```js
const overrides = stripSrcFields(JSON.parse(raw));
return mergeContent(defaults, overrides);
```

Do not export `stripSrcFields`. Do not change `mergeContent`.
