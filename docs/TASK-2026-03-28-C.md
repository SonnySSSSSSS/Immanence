## Goal

`NavigationSection` is the only main section component still eagerly imported in `App.jsx`. The other four sections (`PracticeSection`, `HomeHub`, `WisdomSection`, `ApplicationSection`) are all lazy-loaded. `NavigationSection` is therefore a candidate for the same route-level lazy split already used by the other main sections.

Separately, `NavigationRitualLibrary` is imported at `App.jsx` line 13 but **never rendered in App.jsx**. It is only rendered inside `PracticeSection.jsx` (line ~2530), which is already lazy-loaded. Removing this dead import eliminates unnecessary initial bundle inclusion risk from `RitualSelectionDeck`, `RitualSession`, and their dependencies.

## Implementation Plan

### Step 1 — Convert NavigationSection to lazy import

Remove line 12:
```js
import { NavigationSection } from "./components/NavigationSection.jsx";
```

Add a lazy declaration after the existing lazy declarations (after line ~11):
```js
const NavigationSection = lazy(() =>
  import("./components/NavigationSection.jsx").then(m => ({ default: m.NavigationSection }))
);
```

This matches the exact pattern used for `PracticeSection`, `HomeHub`, `WisdomSection`, and `ApplicationSection`.

### Step 2 — Remove dead NavigationRitualLibrary import

Delete line 13:
```js
import { NavigationRitualLibrary } from "./components/NavigationRitualLibrary.jsx";
```

No other changes needed — the component is not referenced elsewhere in App.jsx.

### Step 3 — Wrap render site in Suspense

At line ~150, change:
```jsx
{section === "navigation" && !hideCards && <NavigationSection ... />}
```

To:
```jsx
{section === "navigation" && !hideCards && (
  <Suspense fallback={<div />}>
    <NavigationSection ... />
  </Suspense>
)}
```

Use `<div />` as the fallback — matching the blank fallback pattern already used for PracticeSection and other sections in `App.jsx`. This is consistency with the existing pattern, not a recommendation for loading UI design. Do not invent a spinner or skeleton for this task. `lazy` and `Suspense` are already imported from React at line 2.

**No other changes to App.jsx. No changes to any other file.**
