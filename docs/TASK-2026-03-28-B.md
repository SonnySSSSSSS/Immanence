## Goal

This is a bounded a11y pass across five files with two change patterns:

1. Backdrop dismiss target semantics
2. AuthGate input labels

## Implementation

### 1. Backdrop dismiss targets

Replace each standalone clickable backdrop element with a keyboard-operable dismiss control.

Preferred pattern:

```jsx
<button
  type="button"
  className="absolute inset-0 bg-black/80"
  onClick={onClose}
  aria-label="Dismiss dialog"
  style={{ border: 'none', padding: 0, backgroundColor: 'transparent' }}
/>
```

Use the existing backdrop class tokens already present in each file rather than normalizing all of them to one example string.

Preserve each file's existing handler:

- **BenchmarkInput** → `onClose`
- **CheckpointReview** → `onClose`
- **ModeSwitchDialog** → `onClose`
- **DevPanel** → existing inline state setter
- **AuthGate prompt** → existing prompt dismiss handler

**Important constraint:**

- Do not suppress focus styles unless an existing equivalent focus indicator is already present.
- Resting-state visuals must remain unchanged.
- A keyboard focus ring appearing when the backdrop is focused is acceptable.

**Intentional tradeoff:**

This task makes the backdrop itself keyboard-operable and therefore may introduce an extra tab stop. Do not add focus trap, Escape handling, or broader modal semantics in this task.

### 2. AuthGate labels

Add visually hidden labels to all four AuthGate inputs using existing Tailwind `sr-only`.

Example pattern:

```jsx
<label className="sr-only" htmlFor="auth-email">Email</label>
<input id="auth-email" ... placeholder="Email" />
```

Requirements:

- Unique `id` for each input
- No duplicate `id` values between signup form and name prompt modal
- Do not remove placeholders
- Do not add `<fieldset>` or `<legend>`
- Do not change layout
