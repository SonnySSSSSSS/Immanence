## Goal

Make the first actionable Day 1 scheduled practice complete end-to-end:

`Today's Practice card -> launch correct practice -> complete session -> show SESSION COMPLETE -> persist curriculum/path completion state`

---

## Problem Statement

The student flow now reaches:

- authenticated state,
- persisted `Student` mode,
- initiation setup,
- Day 1 anchored to today when valid.

The next unverified boundary is the actual practice handoff. The chain is not complete until a student can use the first scheduled Day 1 card to launch the intended practice, finish it, receive completion feedback, and have the result reflected back into the curriculum/path state.

---

## Wrong Behavior To Fix

**Current risk / target wrong behavior**

- HomeHub shows an actionable Day 1 slot,
- but clicking it may fail to launch the intended practice, launch with stale or missing path context, fail to show `SESSION COMPLETE`, or fail to persist leg completion back to the curriculum/path state.

**What this task is fixing**

- Any break in the single runtime chain:
  - `DailyPracticeCard` start action
  - transient launch context handoff
  - `PracticeSection` runtime start
  - completion summary display
  - session recording
  - curriculum leg completion persistence

**Exact real process problem**

- A student can be onboarded and see Day 1, but if the first actual practice cannot be completed and recorded from the scheduled card, the product is still not student-ready.

---

## Expected Right Behavior

When Day 1 shows an actionable first slot:

- clicking the slot opens the correct practice,
- the launched practice carries the correct active-path context,
- completing the session shows `SESSION COMPLETE`,
- the related curriculum leg is marked complete,
- the active path/curriculum state reflects the completion after return or reload.

---

## Real Process Problem This Fix Addresses

This addresses the real handoff from onboarding into actual use:

- setup is no longer enough,
- the student must be able to do the first assigned practice immediately,
- they need explicit completion feedback,
- and the app must remember that completion.

If this handoff fails, the student experience still stops at the first real use of the app.
