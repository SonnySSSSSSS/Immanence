# TASK-2026-03-28-E: 375px HomeHub Mobile Audit (No Implementation)

**Created:** 2026-03-28
**Recommended Executor:** Human + Claude Code
**Scope:** Audit HomeHub at 375px viewport width. Identify touch-target failures, layout overflow, and clipping. Report findings only — no code changes in this task.
**Flags:** `AUDIT ONLY` `NO CODE CHANGES`

---

## Goal

HomeHub is the primary hub surface after login. It composes multiple child components (`DailyPracticeCard`, `HubCardSwiper`, `SimpleModeButton`, `CurriculumHub`, `AvatarV3`) whose touch-target sizes and layout behavior at 375px are unknown without runtime inspection.

This task measures the current state and produces a findings report. It does not implement fixes, because the actual tappable controls are rendered by child components — any corrections would require expanding scope beyond `HomeHub.jsx` into files like `DailyPracticeCard.jsx`, `HubCardSwiper.jsx`, or `SimpleModeButton.jsx`. That scope expansion needs its own spec.

---

## Audit Procedure

1. Run `npm run dev`
2. Sign in and navigate to HomeHub
3. Open Chrome DevTools → toggle device toolbar → set to 375px width
4. Screenshot the full HomeHub view at 375px

For each of the following, measure computed height and report pass/fail against the repo's 44px baseline:

| Element | Owner Component | Expected Location |
|---------|----------------|-------------------|
| Daily practice button | `DailyPracticeCard.jsx` | Top of hub |
| Section nav buttons | `HomeHub.jsx` or `SideNavigation.jsx` | Hub navigation |
| Hub card swiper controls | `HubCardSwiper.jsx` | Card carousel |
| Avatar tap area | `AvatarV3.jsx` | Avatar header |
| Curriculum action buttons | `CurriculumHub.jsx` | Curriculum card |
| Any other tappable control visible at 375px | Various | As found |

Additionally check:

- [ ] HomeHub rail (`--ui-rail-max: min(430px, 94vw)`) fits without horizontal scroll at 375px
- [ ] Hub card swiper is swipeable and cards don't clip
- [ ] Avatar area does not overflow
- [ ] No text truncation that hides actionable content

---

## Deliverable

A findings report containing:

1. Screenshot of HomeHub at 375px
2. For each measured element: component file, computed height, pass/fail
3. List of elements that need correction, grouped by owner component file
4. Recommendation for which files would need to be in the allowlist for a follow-up implementation spec

---

## Constraints

1. **No code changes.** This is an audit-only task.
2. Do not modify any file.
3. Do not create a follow-up implementation spec — only report findings. The human reviewer decides next steps.

## Success Criteria
