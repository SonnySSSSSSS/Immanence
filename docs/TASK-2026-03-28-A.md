# TASK-2026-03-28-A: Student Mode — Curriculum-Gated Practice Navigation

**Created:** 2026-03-28
**Recommended Executor:** Claude Code / Codex
**Scope:** Allow students in guided/student mode to navigate to the Practice section freely, but restrict the visible practice selector to only the practices permitted by the active curriculum path (`allowedPractices`). If no path is active, the Practice button remains locked.
**Flags:** `IMPLEMENT UNCOMMITTED` `NO COMMIT WITHOUT HUMAN APPROVAL`

---

## Goal

In student (guided) mode, the Practice nav button is currently silently swallowed — clicking it does nothing unless a `practiceLaunchContext` is set (i.e., a curriculum card was tapped). This is too restrictive. Students should be able to enter the Practice section freely and practice what their curriculum allows, but not have access to practices outside the curriculum's `allowedPractices` list.

The `allowedPractices` field already exists in the path schema (`src/data/navigationData.js`) and is populated for the `initiation` path (`["breath", "circuit"]`). It is not yet consumed anywhere. This task wires it up end-to-end.

**No sequencing.** Implement this feature bundle completely within the allowlist. Do not defer mode checks, selector filtering, or correction behavior to follow-up tasks.

---

## Problem Statement

1. **Navigation blocked**: `handleSectionSelect` in `App.jsx` blocks `practice` navigation in guided mode unless `practiceLaunchContext` is set. The HomeHub practice button already passes `{ forceStudentNavigation: true }` — this option is not honored for `practice` (only for `navigation`).

2. **No practice filtering**: The local `PracticeSelector` in `PracticeSection.jsx` renders a hard-coded list of all 6 practices regardless of the active path or user mode.

3. **No fallback when current practice is not allowed**: If the user's saved `practiceId` is not in `allowedPractices`, the section silently loads the disallowed practice.

---

## State Model (Three Semantic States)

`allowedPracticeIds` must represent three distinct states — not two. Collapsing them causes subtle behavioral failures.

| State | `allowedPracticeIds` value | Selector behavior |
|---|---|---|
| Not student/guided mode | `null` | Render full list. Unchanged from current behavior. |
| Student/guided mode, no active path | `'locked'` (string sentinel) | Render **nothing** (`null`). Navigation is blocked upstream, so this state should not be reachable — but if it is, the selector must not expose any practices. |
| Student/guided mode, active path, no `allowedPractices` defined | `null` | Render full list. **Fail-open policy** (explicit): missing config means unrestricted. Future paths that omit `allowedPractices` get full access until explicitly restricted. Deliberate operational choice. |
| Student/guided mode, active path, `allowedPractices` defined | `string[]` | Render filtered list. Only listed practice IDs visible. |

**The `'locked'` sentinel and `null` (unrestricted) must not collapse into the same value.** State 2 (locked) and state 3 (unrestricted) are semantically different: locked means the user should not be here at all; unrestricted means they are here and the config allows everything.

---

## Architectural Constraints (Read Before Implementing)

**`App.jsx` does not import `useNavigationStore`.** Verify this before writing any code. This means `activePath` is not available in `App.jsx`'s `handleSectionSelect` callback — adding it requires a new subscription.

**`activePath` in `navigationStore` is an object**, not a string. Its shape is `{ activePathId, status, ... }`. The path ID is at `activePath?.activePathId`. Use this field — not the parent object — as the presence check for "has an active path." Using the parent object alone (`activePath` truthy) risks passing with a malformed or stale object missing `activePathId`.

**Explorer/full mode can coexist with a non-null `activePath`** in store state (a user can retain a stored path from a previous student-mode session). This means derivation logic in `PracticeSection.jsx` must gate on **both** `accessPosture === 'guided'` **and** `activePath`, not on `activePath` alone. Filtering on path presence without the mode check would incorrectly restrict explorer/full mode users who have a stored path.

## Implementation Plan

### Step 1 — `src/App.jsx`: subscribe to `activePath`, honor `forceStudentNavigation` for practice

Add the `useNavigationStore` import and subscribe to `activePath`:

```js
import { useNavigationStore } from './state/navigationStore.js';
// ...
const activePath = useNavigationStore((s) => s.activePath);
```

In `handleSectionSelect`, change the `practice` branch. The outer `if (accessPosture !== 'guided')` branch already returns early for non-guided modes, so this block is only reachable in guided mode. Gate explicitly on `activePathId` — not the parent object — to avoid passing with a stale or malformed path:

```js
if (section === 'practice') {
  if (practiceLaunchContext) {
    setActiveSection('practice');
    return;
  }
  if (options?.forceStudentNavigation === true && activePath?.activePathId) {
    setActiveSection('practice');
  }
  return;
}
```

Add `activePath` to the `useCallback` dependency array.

**No other changes to `App.jsx`.**

### Step 2 — `src/components/PracticeSection.jsx`: derive `allowedPracticeIds`

`PracticeSection.jsx` already reads `activePath` (line ~355, object shape). Verify whether `accessPosture` is already subscribed. If not, add:

```js
const accessPosture = useUserModeStore((s) => s.accessPosture);
```

`useUserModeStore` is already imported — do not add a duplicate import.

After the `activePath` subscription, derive `allowedPracticeIds`:

```js
const allowedPracticeIds = useMemo(() => {
  if (accessPosture !== 'guided') return null;
  if (!activePath?.activePathId) return 'locked';
  const pathDef = getPathById(activePath.activePathId);
  const allowed = pathDef?.tracking?.allowedPractices;
  // Fail-open: missing or empty allowedPractices → unrestricted (null)
  return Array.isArray(allowed) && allowed.length > 0 ? allowed : null;
}, [accessPosture, activePath]);
```

The semantic dependency is `accessPosture` and `activePath?.activePathId`. The dep array uses the full `activePath` object because React memoization requires a stable reference — not a derived field — but the derivation logic only acts on `activePathId`. If `activePath` identity changes without `activePathId` changing, the memo re-runs but produces the same value, which is harmless.

### Step 3 — `src/components/PracticeSection.jsx`: guarded auto-correct effect

Add this effect near the other `practiceId`-correction effects (~line 688). It resets to the first allowed practice when the saved `practiceId` is outside the allowed set. Guard against all active and pending session lifecycle states — the intent is "only fire when the section is idle and the selector is visible," even though visibility is not directly queryable.

```js
useEffect(() => {
  if (!Array.isArray(allowedPracticeIds)) return; // null (unrestricted) or 'locked'
  if (isRunning || activePracticeSession) return;  // replace second var after verifying (see below)
  if (!allowedPracticeIds.includes(practiceId)) {
    setPracticeId(allowedPracticeIds[0]);
  }
}, [allowedPracticeIds, practiceId, isRunning, activePracticeSession]);
```

**Before writing this effect:** verify the exact variable names for `isRunning` and the active/pending session state variable in `PracticeSection.jsx`. The second guard (`activePracticeSession` in the example) must represent a variable that is truthy during both an active session **and** any transition state (launching, tearing down). If the actual variable only covers one of those, find the correct combination. Do not omit the guard. `setPracticeId` is a Zustand store setter and is stable by contract — it may be omitted from deps.

### Step 4 — `src/components/PracticeSection.jsx`: filter local `PracticeSelector`

The inline `PracticeSelector` (line ~218) has a hard-coded list. Change it to accept `allowedPracticeIds` and apply the three-state behavior. **Preserve the existing canonical order exactly — only remove disallowed items, do not reorder.**

```js
function PracticeSelector({ selectedId, onSelect, allowedPracticeIds }) {
  const items = useMemo(() => {
    // 'locked' sentinel → render nothing
    if (allowedPracticeIds === 'locked') return [];
    const allIds = ['breath', 'integration', 'circuit', 'awareness', 'resonance', 'perception'];
    const visibleIds = Array.isArray(allowedPracticeIds)
      ? allIds.filter(id => allowedPracticeIds.includes(id))
      : allIds; // null → unrestricted
    return visibleIds.map((id) => {
      const p = PRACTICE_REGISTRY[id];
      return { id, label: p.label, rail: getRailColor(id) };
    });
  }, [allowedPracticeIds]);

  if (items.length === 0) return null;

  return (
    <div data-tutorial="practice-selector">
      <CircuitTrainingSelector
        items={items}
        value={selectedId}
        onChange={onSelect}
      />
    </div>
  );
}
```

At the render site (~line 3408), pass the derived value:

```jsx
<PracticeSelector
  selectedId={practiceId}
  onSelect={handleSelectPractice}
  allowedPracticeIds={allowedPracticeIds}
/>
```
